# SFlow v1.1 — Miss Log + Better Prompt Design

**Date:** 2026-05-13
**Status:** Draft, pending user review
**Scope:** Items #5 and #1 from `docs/v1.1-roadmap.md`

## Goal

Two independent improvements shipping in one session:

1. **Miss log (#5)** — Local-only logging of cases where the user clicked an interactive UI element but no SFlow rule matched. CLI tool to analyze the log and identify which apps/buttons need rule improvements.
2. **Better Claude prompt (#1)** — Improve the LLM rule-discovery prompt with title variants and few-shot examples to raise match rate from ~50% to >70%.

Both ship in one PR with separate commits. They target different files (Swift client vs TypeScript backend) and are independent, but thematically related: miss log measures, prompt improves.

## Non-Goals

- ❌ Verify pass (second Claude call to validate generated rules)
- ❌ Menu bar UI / Settings window
- ❌ Toggle to disable miss logging / Clear button
- ❌ Functional remote re-seed mechanism (we add the `version` field as forward-compat, but no client-side version check / refresh logic)
- ❌ Endpoint `/v1/refresh` on backend
- ❌ Server upload of miss data (stays local)

## Part A — Miss Log

### What we log

A "miss" is: the user clicked on an AX element with an **interactive role** (`AXButton`, `AXMenuItem`, `AXMenuButton`, `AXPopUpButton`), and none of the rule layers L0.5–L4 produced a match (i.e., no toast shown).

We do **not** log:
- Clicks on non-interactive elements (text fields, scrollbars, decorative groups)
- Clicks when SFlow is disabled (Enabled toggle off)
- Successful clicks where a rule matched (those are toast events, already logged)

### Event schema (JSONL, one event per line)

Existing `events.jsonl` format gets a `type` field. Two cases:

```json
{"type":"toast","ts":"2026-05-13T17:55:00Z","bundleId":"com.tinyspeck.slackmacgap","shortcutId":"slack-new-message","keys":["meta","k"],"hint":"⌘K","mouseX":120,"mouseY":340}
{"type":"miss","ts":"2026-05-13T17:55:30Z","bundleId":"md.obsidian","role":"AXButton","title":"open quick switcher","desc":"","help":""}
```

**Backward compat:** Pre-existing lines (no `type` field) are treated as `"type":"toast"` by the analyzer.

### Privacy

- Miss events contain only AX metadata of the clicked element (role/title/desc/help) — the same fields already sent to `/v1/discover`.
- No text content, no clipboard data, no screen contents.
- File stays at `~/Library/Application Support/SFlow/events.jsonl`, never leaves the Mac.
- No clear/disable controls in v1.1 — user manages the file manually if desired.

### `SFlow --analyze` CLI report format

Plain text to stdout. Apps ranked by miss count (descending), each followed by top-10 most-missed (role, title) tuples:

```
SFlow Miss Analysis — 2026-05-13
=================================

Obsidian (md.obsidian) — 234 misses
  47x  AXButton  "open quick switcher"
  31x  AXButton  "new note"
  22x  AXMenuItem "Toggle Pin"
  ...

Linear (com.linear) — 67 misses
  19x  AXButton  "Create issue"
  ...

(2 more apps with fewer than 10 misses each)

Total: 314 misses across 4 apps. Toast events: 89.
```

App display name resolution: from existing `bundleIdToName` mapping if available; otherwise raw bundleId.

## Part B — Better Prompt + Auto Re-seed

### Prompt changes (`backend/src/prompt.ts`)

Three modifications to `buildSystemPrompt()`:

1. **New "TITLE VARIANTS" section** — explicit instruction that each rule's `titles` array must include 3–5 variants:
   - Verb-led English (e.g., "Open Quick Switcher")
   - Noun-only English (e.g., "Quick Switcher")
   - Verbose form (e.g., "quick switcher button")
   - Common localizations only when confident (existing rule preserved)

2. **Few-shot examples** — 2–3 concrete pairs of `(AX skeleton fragment → ideal rule)`. Stored in new file `backend/src/prompt-examples.ts` for clarity, imported into the prompt builder.

3. **Schema update** — rule objects must include `"version": 1`. Always 1 in v1.1; defensive normalization on backend side if Claude omits it or returns garbage.

### `version` field (dormant forward-compat)

- Backend adds/normalizes `version: 1` on every rule before responding.
- TypeScript `Rule` type in `backend/src/types.ts` gets `version: number`.
- Swift `Rule` Codable in `SFlow/RuleCache.swift` gets `version: Int?` (optional, defaults to 1 for backward compat with existing `bundled.json` entries).
- Client does **not** check `version` against backend — that's v1.2 work. Field is stored and ignored for now.

### Auto re-seed (`SFlow --reseed <bundleId>` / `--reseed-all`)

New CLI mode in the same binary. When invoked, skips GUI setup and runs `Reseeder.run(args:)`.

- `--reseed <bundleId>` — single app, useful for retry on one failure
- `--reseed-all` — loops through the hardcoded verified-app list (see below)

Both share the same pre-flight checks and per-app logic. Difference is only the loop.

**Hardcoded verified-app list** in `Reseeder.swift`:

```swift
let VERIFIED_APPS = [
    "com.tinyspeck.slackmacgap",   // Slack
    "md.obsidian",                  // Obsidian
    "com.linear",                   // Linear
    "com.todesktop.230313mzl4w4u92" // Cursor
]
```

**Pre-flight checks** (fatal if any fail):
- GUI SFlow not running (`NSRunningApplication` lookup) — conflict with event tap
- AX permissions granted — required for skeleton capture

**Per-app loop:**
1. Check installation: `NSWorkspace.shared.urlForApplication(withBundleIdentifier:)`. If nil, skip with warning, continue.
2. Launch: `NSWorkspace.shared.openApplication(...)`. Wait until AX responsive (poll AXUIElementCopyAttributeValue every 200ms, max 10s timeout).
3. Bring frontmost.
4. Capture AX skeleton + menu bar — reuse the existing discovery capture code path in `DiscoveryClient` (exact method name to be confirmed during implementation; the logic that today runs on first click in an unknown app).
5. POST `/v1/discover` with payload. Retry once on network failure.
6. On success: write response to `cache/<bundleId>.json`.
7. On parse failure: dump raw response to `/tmp/sflow-reseed-error-<bundleId>.json`, skip app.
8. Do **not** quit the target app — user may have unsaved work.
9. Print progress: `✓ Slack: 12 rules (3 new vs cached)`.

**After loop:** print summary, suggest `scripts/promote-to-bundled.sh`.

### `scripts/promote-to-bundled.sh`

Merges all `cache/<bundleId>.json` files into the shipping `bundled.json`. Prints `git diff bundled.json` so user can review before committing. Does NOT auto-commit.

### Backup of `bundled.json`

Before any `--reseed-all` run, `Reseeder` copies the current `bundled.json` to `bundled.json.bak.<ISO8601-timestamp>` in the same directory. Added to `.gitignore`. Two-level revert if new rules are worse than old:

- **Fast:** `mv bundled.json.bak.<ts> bundled.json` + rebuild
- **Deep:** `git reset --hard pre-v1.1-misslog-prompt-2026-05-13`

### Testing

- `buildSystemPrompt()` — snapshot test (string match) to catch regressions
- Codable round-trip for `Rule` with and without `version` field (backward compat)

- End-to-end success metric: manually click 10 known buttons in each of the 4 apps, count toasts. Target ≥70% (current ~50%).

## Rollout Plan

In sequence, each step is its own commit/push:

1. **Safety checkpoint:** `git tag pre-v1.1-misslog-prompt-2026-05-13 && git push origin pre-v1.1-misslog-prompt-2026-05-13`
2. **Miss log (Swift)** — EventLogger refactor, ClickWatcher hook, Analyzer, main.swift dispatch, sflow-analyze wrapper. Build + manual smoke test.
3. **Better prompt (backend)** — prompt.ts + prompt-examples.ts + types.ts. Snapshot test. Deploy via `wrangler deploy`.
4. **Auto re-seed (Swift)** — Reseeder.swift + sflow-reseed wrapper + promote script. Build.
5. **Run `sflow-reseed-all`** — auto-loop through 4 apps, fresh rules land in `cache/`. Review `git diff cache/`.
6. **Promote to bundled.json** — `scripts/promote-to-bundled.sh`. Review `git diff bundled.json`. Commit.
7. **Manual eval** — click 10 known buttons per app, count toasts. Confirm ≥70%.
