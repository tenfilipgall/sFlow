## 1. Problem

Klikam w Gmaila w przeglądarce Comet → SFlow widzi `bundleId="ai.perplexity.comet"`
→ stosuje reguły dla **Comet** (browser chrome: tabs, address bar). **Reguły
Gmaila (j/k nav, c compose, e archive) nie istnieją w SFlow.**

To samo dla **każdej** apki web:
- Slack web (`app.slack.com`)
- Notion web (`notion.so`)
- Linear web (`linear.app`)
- Gmail (`mail.google.com`)
- GitHub (`github.com`)
- Figma (`figma.com`)
- Google Docs / Sheets / Slides
- Trello, Asana, Monday.com
- Jira, Confluence

**Skala:** power-user 2-4h dziennie w przeglądarce. Możliwe że **30-40%
kliknięć usera** trafia w web content. Dziś SFlow ich nie pokrywa.

## 2. Mechanizm — 3 etapy

### Etap A: URL detection w drzewie AX (~2h, **empiryczne**)

**Hipoteza 1:** Chromium eksponuje `AXURL` na `AXWebArea` w drzewie AX.

**Test empiryczny dla Filipa (do wykonania ZANIM zaczniemy implementację):**

Otwórz Comet/Chrome z Gmail otwartym. W Terminalu uruchom:

```swift
// Pseudo — skrypt do napisania w `scripts/sflow-probe-ax-url.swift`
import ApplicationServices
import AppKit

let app = NSWorkspace.shared.frontmostApplication!
let axApp = AXUIElementCreateApplication(app.processIdentifier)
AXUIElementSetAttributeValue(axApp, "AXManualAccessibility" as CFString, kCFBooleanTrue)

// Walk drzewa, szukaj AXWebArea
func walk(_ el: AXUIElement, depth: Int) {
    var roleRef: AnyObject?
    AXUIElementCopyAttributeValue(el, kAXRoleAttribute as CFString, &roleRef)
    let role = roleRef as? String ?? ""

    if role == "AXWebArea" {
        // Sprawdź wszystkie attrybuty
        var attrs: CFArray?
        AXUIElementCopyAttributeNames(el, &attrs)
        print("AXWebArea found at depth \(depth). Attributes:")
        for attr in (attrs as? [String]) ?? [] {
            var val: AnyObject?
            AXUIElementCopyAttributeValue(el, attr as CFString, &val)
            print("  \(attr) = \(String(describing: val).prefix(100))")
        }
    }

    guard depth < 10 else { return }
    var childrenRef: AnyObject?
    AXUIElementCopyAttributeValue(el, kAXChildrenAttribute as CFString, &childrenRef)
    for child in (childrenRef as? [AXUIElement]) ?? [] {
        walk(child, depth: depth + 1)
    }
}
walk(axApp, depth: 0)
```

**Oczekiwane outputs:**
- ✅ `AXURL = "https://mail.google.com/..."` — perfect, używamy
- ✅ `AXDocument = "https://..."` — alternatywa
- ❌ Pusty AXURL / nie ma AXWebArea — fallback do Hipotezy 2

**Hipoteza 2 (fallback):** Ekstrakcja domeny z `AXTitle` okna.

Okno Comet/Chrome ma title typu `"Inbox (1,234) - filip@gmail.com - Gmail"`.
Regex: `(?: - |^| – )([A-Z][a-z]+(?: [A-Z][a-z]+)?)\s*$` — wyciąga "Gmail",
"Notion", "Slack", "GitHub". Mapping: `"Gmail"` → `web:gmail.com`.

**Słabość Hipotezy 2:** zależy od formatu title (Slack zmienia), wielojęzyczne,
nieprzewidywalne.

**Decyzja:** spróbować H1 najpierw. Jeśli AXURL dostępne → użyć. Jeśli nie
→ H2 jako fallback z whitelistą domen.

### Etap B: Pseudo-bundleId infrastructure (~2h)

Wprowadzić **dwa-warstwowe bundleId** w `ClickWatcher.handleMouseDown`:

```swift
let appBundleId = frontmost.bundleIdentifier  // np. "ai.perplexity.comet"
let webDomain = extractWebDomain(from: axApp, fallback: appBundleId)
let effectiveBundleId: String

if let domain = webDomain {
    effectiveBundleId = "web:\(domain)"  // np. "web:mail.google.com"
} else {
    effectiveBundleId = appBundleId
}

// Reszta pipeline'u używa effectiveBundleId
```

`RuleCache.match` dostaje `effectiveBundleId` → wybiera reguły
`web:mail.google.com.json` zamiast `ai.perplexity.comet.json`.

**Fallback chain:**
- 1. Próbuj `web:{domain}` reguły
- 2. Jeśli nie znalazł → próbuj `appBundleId` reguły (browser chrome)
- 3. Jeśli nie znalazł → universal heuristics (L4)

To zachowuje pokrycie chrome (tabs, address bar) **i** dodaje web app
pokrycie.

## 3. Test-driven kroki (po empirycznym test §2A)

### 3.1. URL extraction helper

**Nowy plik:** `SFlow/WebDomainExtractor.swift` (~80 LOC).

```swift
enum WebDomainExtractor {
    /// Returns domain portion of URL found in AXWebArea, or nil if app is not
    /// a browser or URL is unreadable. Empirically validated on Comet/Chrome.
    static func extract(from axApp: AXUIElement, bundleId: String) -> String? {
        // Only run for known browsers — skip overhead for native apps
        guard isBrowserBundle(bundleId) else { return nil }

        // Walk AX tree (max depth 5) looking for AXWebArea with AXURL
        if let url = findAXURL(in: axApp, maxDepth: 5),
           let domain = canonicalDomain(from: url) {
            return domain
        }
        return nil
    }

    static func canonicalDomain(from urlString: String) -> String? {
        // "https://mail.google.com/u/0/inbox" → "mail.google.com"
        // "http://www.github.com/" → "github.com" (strip www.)
        guard let url = URL(string: urlString),
              let host = url.host?.lowercased() else { return nil }
        return host.hasPrefix("www.") ? String(host.dropFirst(4)) : host
    }

    static let browserBundles: Set<String> = [
        "ai.perplexity.comet",
        "com.google.chrome", "com.google.chrome.canary",
        "com.microsoft.edgemac",
        "company.thebrowser.Browser", // Arc
        "com.brave.Browser",
        "com.operasoftware.Opera",
        "org.mozilla.firefox",
        "com.apple.Safari",
    ]

    static func isBrowserBundle(_ id: String) -> Bool {
        browserBundles.contains(id)
    }
}
```

## 6. Ryzyka

### Ryzyko 1: AXURL nie istnieje

**Diagnoza:** AppKit AX API nie ma `kAXURLAttribute` w nagłówkach.
Chromium może eksponować custom string ale niespotykane.

**Mitigacja:** Hipoteza 2 (title parsing) ma działać. Whitelist top-20
domen mapowanych z title fragments ("Gmail" → mail.google.com).

### Ryzyko 2: SPA przewija strony bez zmiany title

Gmail switching folder nie zmienia title. SFlow nie wie kiedy domena
się zmieniła w obrębie SPA.

**Mitigacja:** akceptujemy. To wciąż jest **Gmail** więc reguły działają.
Per-folder subtle differences (np. archive nie ma sensu w sent folder)
to znikomy issue.

### Ryzyko 3: Filename z `:` na FS

Pseudo-bundleId `web:gmail.com` jako nazwa pliku. macOS HFS+/APFS akceptuje
`:` w API ale **w GUI Finder pokazuje jako `/`**. Może być problem dla
manual edit.

**Mitigacja:** użyj alternatywnego separatora w nazwach plików:
`bundled/web__mail.google.com.json` (dwa underscore). Mapping at load/save
time.

### Ryzyko 4: Privacy — czytamy URL z drzewa AX

Wcześniej SFlow nigdy nie wiedział co user przegląda. Teraz wiemy że
jest w `mail.google.com`. To kompresja prywatności.

**Mitigacja:**
- URL **NIE** trafia do `events.jsonl` jako raw — tylko pseudo-bundleId
  (`web:mail.google.com`)
- Path/query parameters URL nigdy nie są ekstraktowane (tylko host)
- W Settings dodać toggle "Allow SFlow to detect web apps" (default ON)
- Disclose w product-vision section 5e (Privacy)
